# 多平台消息格式适配指南

## 通用消息结构

所有平台消息统一抽象为以下结构：

```
Message {
  platform: string,        // taobao|pdd|jd|douyin
  shopId: string,          // 店铺ID
  buyerId: string,         // 买家ID（脱敏）
  buyerNick: string,       // 买家昵称
  messageType: string,     // text|image|order_query|complaint
  content: string,         // 消息正文
  timestamp: string,       // ISO 8601
  orderContext: {          // 关联订单（可选）
    orderId: string,
    status: string,        // pending|shipped|delivered|completed
    logistics: object,
    product: object
  }
}
```

## 各平台适配要点

### 淘宝/天猫（千牛）
- 消息接口：千牛开放平台消息推送
- 客服状态：在线/挂起/离线，通过千牛状态同步
- 自动回复：通过千牛机器人插件实现
- 订单查询：通过淘宝开放平台TOP接口
- 敏感操作需用户授权

### 拼多多
- 消息接口：多多客服消息推送
- 自动回复：多多客服机器人
- 注意：拼多多对回复时效要求更严格（5分钟内）
- 不支持发送外链和二维码

### 京东
- 消息接口：京东咚咚客服
- JIMI机器人可辅助应答
- 对商家资质审核较严

### 抖音/抖店
- 消息接口：抖店开放平台
- 支持飞鸽客服系统
- 消息类型：文本/图片/视频/商品卡片
- 3分钟回复率影响店铺评分

### 快手/快店
- 消息接口：快手小店客服
- 类似抖店，文字+图片为主

## 应答时效标准

| 平台 | 首次响应 | 平均响应 | 影响指标 |
|------|---------|---------|----------|
| 淘宝 | 3分钟 | 保持在线 | 店铺体验分 |
| 拼多多 | 5分钟 | 3分钟 | 回复率考核 |
| 京东 | 3分钟 | 保持在线 | 咚咚评分 |
| 抖音 | 3分钟 | 2分钟 | 3分钟回复率 |
| 快手 | 3分钟 | 保持在线 | 客服体验分 |

## 多店铺统一管理

- 每个店铺独立维护应答上下文
- 共享FAQ知识库，差异化优惠策略
- 统一数据看板：按店铺维度拆分客服数据
- 消息路由：买家消息携带shopId，自动匹配对应店铺策略
