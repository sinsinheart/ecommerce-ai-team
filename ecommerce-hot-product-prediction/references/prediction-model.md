# 爆款预测算法模型

## 六大维度评分模型

### 维度1：热度涨幅（权重25%）
```
热度涨幅得分 = (当前周期热度 - 上周期热度) / 上周期热度 × 100

评分标准：
- 涨幅 >100% → 25分
- 涨幅 50-100% → 20分
- 涨幅 20-50% → 15分
- 涨幅 0-20% → 10分
- 涨幅 <0 → 5分
```

### 维度2：销量增速（权重25%）
```
销量增速得分 = (30日销量 - 前30日销量) / 前30日销量 × 100

评分标准：
- 增速 >80% → 25分
- 增速 40-80% → 20分
- 增速 10-40% → 15分
- 增速 0-10% → 10分
- 增速 <0 → 5分
```

### 维度3：用户讨论度（权重15%）
```
讨论度 = (评论数 + 分享数 + 收藏数) / 曝光量 × 100

评分标准：
- 讨论度 >8% → 15分
- 讨论度 5-8% → 12分
- 讨论度 2-5% → 9分
- 讨论度 1-2% → 6分
- 讨论度 <1% → 3分
```

### 维度4：同款店铺数（权重10%）
```
竞争度 = 同款在售店铺数 / 品类总店铺数 × 100

评分标准（越少越好）：
- 同款店铺 <50家 → 10分
- 同款店铺 50-200家 → 8分
- 同款店铺 200-500家 → 6分
- 同款店铺 500-1000家 → 4分
- 同款店铺 >1000家 → 2分
```

### 维度5：市场空白度（权重15%）
```
空白度 = 搜索需求量 / 在售商品数

评分标准：
- 供需比 >5:1 → 15分（高需求低供给）
- 供需比 2:1-5:1 → 12分
- 供需比 1:1-2:1 → 9分
- 供需比 0.5:1-1:1 → 6分
- 供需比 <0.5:1 → 3分（供过于求）
```

### 维度6：季节适配性（权重10%）
```
根据当前日期判断季节匹配度：
- 完全匹配当前季节需求 → 10分
- 即将进入适配季节（30天内）→ 8分
- 适配季节已过但仍有需求 → 5分
- 反季节/无季节关联 → 3分

特殊节点加成：双11(+2)、618(+2)、年货节(+2)、情人节(+2) 每月额外+1
```

## 综合评分公式

```
总评分 = Σ(各维度得分 × 权重)

结果分级：
- 🔥🔥🔥 85-100分 → 高概率爆款，建议立即准备上架
- 🔥🔥   70-85分  → 潜力爆款，建议优先关注
- 🔥     55-70分  → 值得关注，持续观察趋势
- ⚪     <55分    → 暂时不推荐
```

## 爆款分类逻辑

### 短期爆款判定
```
条件：热度涨幅>100% AND 近7天出现 AND 历史无稳定销量
特点：事件驱动，爆发快消退也可能快
策略：快速跟进，小批量试水，设置止盈止损线
```

### 长效趋势款判定
```
条件：销量增速>40% AND 连续30天稳定增长 AND 用户讨论度>5%
特点：需求持续增长，生命周期长
策略：重点投入，完善SKU，长期运营
```

### 季节性爆款判定
```
条件：季节适配性≥8分 AND 去年同季有销量高峰记录
特点：周期性强，可预测
策略：提前30天备货，旺季前上架
```

### 蓝海潜力款判定
```
条件：空白度≥12分 AND 同款店铺<200 AND 热度涨幅>20%
特点：竞争小需求大，适合中小商家
策略：抢先上架占位，建立先发优势
```

## 预测准确率迭代

```
准确率 = 命中爆款数 / 总预测爆款数

迭代规则：
- 准确率 <60% → 调整权重（降低热度涨幅，提升销量增速）
- 误报率 >40% → 增加数据验证轮次
- 某品类持续低准确率 → 标记品类异常，人工复核
```
