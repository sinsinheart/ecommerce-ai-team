# 售后自动化处理

自动化管理电商售后全流程，从退换货审核到物流追踪到异常订单拦截，减少人工操作，降低售后损耗。

## 触发场景

- 买家发起退货/退款申请需要审核
- 售后订单物流需要同步追踪
- 检测到异常售后行为需要预警
- 售后时效即将超时需要提醒
- 商家需要查看售后处理统计

## 核心工作流

### 1. 退货退款自动审核

```
Task Progress:
- [ ] 获取待审核售后申请列表
- [ ] 按审核规则自动处理：
  - [ ] 自动通过：符合7天无理由 + 未拆封 + 买家信誉良好
  - [ ] 自动拒绝：超出时效 + 已使用/影响二次销售 + 退货理由不符
  - [ ] 人工复核：高价值商品 + 买家多次售后 + 举证材料存疑
- [ ] 自动生成审核结果与理由
- [ ] 推送退货地址和注意事项
```

### 2. 审核规则配置

**自动通过条件（可配置）：**
- 订单在售后期内（默认签收后7天内）
- 退货原因属于「7天无理由」或「质量问题」
- 买家信誉分 ≥ 平台基准线
- 该买家近30天退货率 < 阈值（默认20%）
- 订单金额 < 高价值阈值（默认500元）

**自动拒绝条件：**
- 超出平台规定售后期
- 商品已拆封且影响二次销售
- 举证材料与退货理由不符

**人工复核升级条件：**
- 订单金额 ≥ 500元
- 买家近30天退货次数 ≥ 3次
- 举证材料不清晰或有矛盾
- 商家自定义高价值商品

### 3. 物流同步与进度追踪

- 买家退货后自动获取退货物流单号
- 定时轮询物流状态（已揽收/运输中/已签收）
- 商家签收后退货自动确认，触发退款
- 物流异常（退回/丢失）自动告警
- 向买家自动推送售后进度通知

### 4. 恶意订单识别

**薅羊毛行为特征：**
- 同一下单地址多次退货
- 退货商品与下单商品明显不符
- 频繁使用不同账号、同一收货地址退货
- 退货理由前后不一致
- 退货率显著高于店铺均值

**处理策略：**
- 标记可疑订单，自动升级人工复核
- 要求额外举证材料
- 严重恶意行为加入店铺黑名单
- 生成异常订单报告上报平台

### 5. 售后统计报表

定期生成售后数据概览：
- 退货率、退款率、纠纷率及环比变化
- 退货原因分布（质量问题/尺码不符/色差/物流/其他）
- 高退货率商品排行
- 恶意订单拦截数量与金额

## 推送配置

审核结果和异常预警可推送至：
- 商家飞书/企业微信群
- 商家邮件通知
- 店铺后台消息中心
